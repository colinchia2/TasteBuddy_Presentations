Project Context: Built as interview preparation to demonstrate hands-on proficiency with the tools and data types used in health and fitness analytics roles. The goal was to work with a real-world wearable dataset (FitBit) alongside clinical records (MIMIC-III) — the same category of data that companies like WHOOP analyze at scale — and deliver a polished, connected BI output from scratch.
What It Is

A cloud data analytics project built to demonstrate end-to-end proficiency with a modern ELT stack: ingesting raw health data, transforming it with advanced SQL inside Snowflake, and surfacing insights through a Looker Studio dashboard.

The analysis covers two real-world datasets — FitBit activity and sleep logs from 30 Kaggle users, and blood biomarker readings from MIMIC-III, an ICU clinical database from Beth Israel Deaconess Medical Center.

SQL techniques include CTEs, window functions, UNION-based pivots, time-offset date joins, Z-score normalization, regression analysis (REGR_R2), and population-level statistical banding.

ELT Pipeline · How It Works

Two raw CSV datasets — FitBit activity/sleep logs and MIMIC-III lab events — are loaded into Snowflake's RAW_HEALTH_DATA schema. A single SQL script then builds 10 analytical views in ANALYSIS_VIEWS, each targeting a specific reporting need. The night-offset join logic (sleep date = activity date + 1 day) is the core relationship between the FitBit tables, enabling accurate activity-to-recovery correlation. Google Looker Studio (now called Google Data Studio) connects live to these views, so any SQL change is reflected in the dashboard immediately — no export step needed.
